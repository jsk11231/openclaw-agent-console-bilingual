# QUICK_START.md - 一页版快速上手

## 这套系统是什么

这是一套 **虫群式组织架构**：
- 一个入口统一接需求
- 简单任务快速闭环
- 复杂任务动态协同
- 结果统一收口
- 系统持续复盘进化

## 核心结构

- **总调度中心**：接需求、做判断、定路由、统一输出
- **快速响应中心**：处理简单任务和标准化任务
- **协同编排中心**：组织复杂任务和多节点协同
- **专业能力中心**：负责各专业方向执行
- **系统进化中心**：负责复盘、纠错、总结和优化

## 控制台怎么进

如果你要直接看系统状态或改配置，用本地控制台：

- 控制台首页：`http://127.0.0.1:8787`
- 配置审计页：`http://127.0.0.1:8787/audit`

控制台里可以做：
- 看各 agent 状态
- 新增 / 编辑 / 删除 agent
- 拖拽调整架构图布局
- 架构图支持真拖线、横纵自动排布、缩放、根子树边界自适应
- 如果结构被拖乱，可点“规范布局”快速恢复到可渲染状态
- 改 skill
- 做配置审计（重复 skill / 重复职责 / 重复 workspace）
- 切换默认 API 模型（保存后自动重启 gateway）

## 任务怎么判断

### A类：简单闭环任务
特征：
- 一步可完成
- 风险低
- 不需要专业协同

去向：
- 总调度中心 / 快速响应中心

### B类：单能力任务
特征：
- 只需要一个专业中心
- 不需要复杂拆解

去向：
- 对应专业能力中心

### C类：复杂协同任务
特征：
- 多中心参与
- 有依赖关系
- 需要拆解、编排、整合

去向：
- 协同编排中心牵头

### D类：复盘优化任务
特征：
- 目标不是交付业务结果，而是改进系统

去向：
- 系统进化中心

## 工具怎么选

默认顺序：
1. **先看能不能直接完成**
2. **有一等工具就优先工具**
3. **需要方法论时再用 skill**
4. **真复杂再 spawn 子代理**

一句话：
**能直接做就别绕路，能用工具就别空想，真复杂再拆。**

## 什么时候调系统进化中心

建议调它的场景：
- 任务失败后
- 用户明确纠错后
- 复杂任务完成后
- 某类问题反复出现时
- 需要沉淀模板或规则时

不建议调它的场景：
- 普通简单问答
- 一步闭环小任务
- 没有新经验的小修小补

## 一句话总纲

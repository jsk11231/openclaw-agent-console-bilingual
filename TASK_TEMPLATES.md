# TASK_TEMPLATES.md - 高频任务模板

## 目标

把高频、重复、标准化任务沉淀成可复用模板，减少每次临时判断成本，让快速响应中心更稳定，让协同编排中心更省力。

## 使用原则

### 原则1：模板服务于路由
模板不是为了让所有任务都机械化，而是为了让高频任务更快落地。

### 原则2：模板优先用于低风险、重复任务
高风险任务仍然需要人工确认和审慎判断。

### 原则3：模板可迭代
任何模板都可以在系统进化中心复盘后被更新。

## 一、通用任务模板

### 1. 简单问答模板
适用：
- 简单解释
- 轻量判断
- 快速建议

结构：
1. 直接回答
2. 必要时补一句原因
3. 如有风险提示则单独说明

### 2. 信息检索模板
适用：
- 查资料
- 查消息
- 查文档
- 查背景信息

结构：
1. 明确检索目标
2. 选取信息源
3. 汇总结果
4. 如有缺口，说明缺口

### 3. 整理归纳模板
适用：
- 材料整理
- 纪要整理
- 清单整理
- 背景梳理

结构：
1. 原始信息收集
2. 分类归并
3. 提炼重点
4. 输出简洁版本

## 二、飞书相关模板

### 4. 日程创建模板
适用：
- 创建会议
- 安排沟通
- 设置时间段

流程：
1. 确认时间
2. 确认参与人
3. 确认地点/会议链接
4. 创建日程
5. 返回摘要

### 5. 提醒/待办模板
适用：
- 个人提醒
- 跟进事项
- 待办管理

流程：
1. 确认事项内容
2. 确认时间
3. 确认提醒方式
4. 创建任务/提醒
5. 返回确认结果

### 6. 文档处理模板
适用：
- 新建文档
- 更新文档
- 整理结构
- 追加内容

流程：
1. 确认目标文档
2. 确认修改模式
3. 执行操作
4. 检查结果
5. 返回变更摘要

### 7. 消息读取模板
适用：
- 查群消息
- 查私聊消息
- 查 thread
- 查指定时间范围消息

流程：
1. 确认会话范围
2. 确认时间条件
3. 获取消息
4. 提炼上下文
5. 输出结论或摘录

## 三、分析类模板

### 8. 轻分析模板
适用：
- 简单比较
- 轻判断
- 快速评估

结构：
1. 结论先行
2. 列出依据
3. 提示不确定性

### 9. 深分析模板
适用：
- 多因素评估
- 投资分析
- 复杂研究
- 需要结构化判断的任务

结构：
1. 问题定义
2. 已知条件
3. 核心判断
4. 风险点
5. 建议动作

## 四、协同编排模板

### 10. 复杂任务拆解模板
适用：
- 多部门协同
- 长链路任务
- 需要分阶段推进的任务

结构：
1. 任务总目标
2. 子任务拆解
3. 每个子任务负责人/能力中心
4. 顺序与依赖
5. 输出整合方式

### 11. 多节点协同模板
适用：
- 需要多个能力中心一起工作

结构：
1. 主问题
2. 子问题列表
3. 需要调用的中心
4. 节点间依赖
5. 最终汇总口径

## 五、复盘类模板

### 12. 轻复盘模板
适用：
- 小错误
- 单次偏差
- 简单任务后复盘

结构：
1. 问题是什么
2. 原因是什么
3. 下次怎么改

### 13. 标准复盘模板
适用：
- 普通复杂任务
- 用户明确反馈
- 需要过程分析

结构：
1. 任务概况
2. 路由是否合理
3. 执行过程
4. 关键问题
5. 改进建议

### 14. 深度复盘模板
适用：
- 高价值任务
- 系统性问题
- 多次重复失败

结构：
1. 背景与目标
2. 架构层判断
3. 路由层判断
4. 执行层判断
5. 系统性问题
6. 长期优化建议

## 六、技术类模板

### 15. 代码/系统排障模板
适用：
- 技术问题定位
- 配置异常
- 自动化流程问题

结构：
1. 现象
2. 环境
3. 排查步骤
4. 根因判断
5. 修复动作
6. 验证结果

### 16. 工具执行模板
适用：
- read / write / edit
- exec / browser
- gateway
- 其他操作型工具

结构：
1. 目标
2. 选择工具
3. 执行动作
4. 检查结果
5. 输出摘要

## 七、模板使用建议

### 1. 高频任务优先模板化
如果某类任务反复出现，优先提炼成模板。

### 2. 模板不是死的
模板可以在系统进化中心复盘后调整。

### 3. 简单任务先用简模板
不要一上来把所有任务都套进深模板。

### 4. 高风险任务保留人工判断
模板只能辅助，不能替代审慎判断。

## 八、模板与节点的关系

| 节点 | 主要使用模板 |
|---|---|
| 快速响应中心 | 简单问答、信息检索、整理归纳、轻分析模板 |
| 协同编排中心 | 复杂拆解、多节点协同模板 |
| 专业能力中心 | 对应领域的专项模板 |
| 系统进化中心 | 轻复盘、标准复盘、深度复盘模板 |

## 九、一句话结论

模板的本质不是“格式”，而是把高频任务做成可复用的认知压缩包：
**越高频、越标准、越低风险的任务，越值得模板化。**

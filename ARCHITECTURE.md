# ARCHITECTURE.md - 虫群式组织架构

## 架构定位

本体系采用虫群式、扁平化、动态协同的组织结构。

核心原则：
- 一个入口统一接收需求
- 简单任务就地闭环
- 复杂任务动态调用多节点协同
- 最终由入口统一收口反馈
- 系统通过进化节点持续复盘、纠错、优化

本体系不强调传统层级转发，而强调按任务进行动态路由和临时编组。

## 总体架构

```text
老板 / 业务方
   ↓
总调度中心
   ├─ 快速响应中心
   ├─ 协同编排中心
   ├─ 专业能力中心
   │    ├─ 事务支持中心
   │    ├─ 信息支持中心
   │    ├─ 分析研判中心
   │    ├─ 策略规划中心
   │    ├─ 行政保障中心
   │    ├─ 家庭支持中心
   │    ├─ 出行支持中心
   │    ├─ 技术支持中心
   │    ├─ 合规风控中心
   │    └─ 文旅项目中心
   └─ 系统进化中心
```

## 节点职责

### 总调度中心
- 统一接收需求
- 判断复杂度、优先级和所需能力
- 决定直接处理、单节点处理或多节点协同
- 汇总结果并统一反馈

### 快速响应中心
- 处理简单任务
- 承接标准化请求
- 进行轻量整理和快速回复
- 完成低风险一步闭环任务

### 协同编排中心
- 拆解复杂任务
- 组织多节点协同
- 设计任务顺序
- 处理依赖、冲突与资源分配
- 汇总复杂任务结果

### 专业能力中心
按需调用，包含：
- 事务支持中心
- 信息支持中心
- 分析研判中心
- 策略规划中心
- 行政保障中心
- 家庭支持中心
- 技术支持中心
- 合规风控中心
- 文旅项目中心

### 系统进化中心
- 复盘任务过程
- 纠正错误与偏差
- 总结成功模式与失败教训
- 优化路由规则
- 沉淀经验、模板与规则

## 路由原则

1. 简单任务优先直接闭环
2. 单一能力优先
3. 复杂任务动态组队
4. 结果统一收口
5. 进化机制走旁路，不劫持主业务链路

## 工作流

```text
需求进入
   ↓
总调度中心判断
   ├─ 简单任务 → 快速响应中心 → 输出
   ├─ 标准任务 → 专业能力中心单节点 → 输出
   ├─ 复杂任务 → 协同编排中心 → 多节点协同 → 输出
   └─ 复盘优化 → 系统进化中心 → 反哺规则
```

## 一句话说明

总调度中心负责判断和调度，快速响应中心负责小事快办，协同编排中心负责复杂组队，专业能力中心负责具体执行，系统进化中心负责让系统越跑越聪明。

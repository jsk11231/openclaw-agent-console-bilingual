## 核心中枢映射

### 1. `dispatch` → 总调度中心
当前系统中的总入口与总路由节点。

主要职责：
- 接收需求
- 判断复杂度
- 路由分发
- 统一收口

### 2. `fast-response` → 快速响应中心
当前系统中的轻任务与标准任务执行节点。

主要职责：
- 处理简单问答
- 承接一步闭环的提醒、跟进、轻事务
- 快速完成低风险闭环任务

说明：
- `fast-response` 负责一步闭环、低状态依赖的轻事务
- 持续跟进、日程/任务/文档类事务由 `admin-support` 承接

### 3. `orchestration` → 协同编排中心
当前系统中的复杂任务编排节点。

主要职责：
- 复杂任务拆解
- 顺序与依赖管理
- 多节点协同组织
- 结果整合

## 专业能力中心映射

### `information` → 信息支持中心
- 检索资料
- 汇总背景信息
- 清洗和结构化信息
- 适合承接 `find-skills`、`content-breakdown` 这类“先找再整理”的任务
- 不负责专门业务策略，只负责信息输入与整理

### `analysis-center` → 分析研判中心
- 分析
- 评估
- 判断
- 研究输出
- 适合承接公司研究、竞品研究、组织/人物研究、评论分析、关系网络分析等任务
- 也承接宏观数据分析（如 `MX_MacroData`）
- 不负责电商增长专职，也不替代交易策略官

### `strategy-center` → 策略规划中心
- 方案设计
- 路径规划
- 节奏安排
- 优先级建议
- 策略总框架与跨中心方案设计
- 适合承接复杂多目标目标拆解、跨中心方案组合、路线图规划

### `admin-support` → 行政保障中心
- 流程推进
- 资源协调
- 日程、任务、文档等持续型事务支持
- 落地支持

### `tech-support` → 技术支持中心
- 技术实现
- 配置调整
- 系统排障
- 自动化支持
- 适合承接 `clawhub`、`skill-creator`、`skill-writer`、`authoring-skills` 这类技能制作与维护任务

### `compliance` → 合规风控中心
- 合规判断
- 风险控制
- 边界审查

### `tourism-center` → 文旅项目中心
- 文旅专项任务
- 机票、酒店、火车票、打车、租车等出行查询与比对
- 行程规划与旅游相关执行支持

### `evolution-center` → 系统进化中心
- 任务复盘
- 纠错总结
- 规则优化
- 经验沉淀
- 适合承接 `self-improvement`、技能/路由/模板优化、历史问题归因与长期规则沉淀
- 不承接日常策略输出，只做优化旁路
